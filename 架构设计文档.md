# 财务自由助手 — 架构设计文档

## 项目定位

AI 驱动的个人财务管理智能体。当前阶段聚焦零钱管理，后续扩展投资建议、税务规划等能力。

## 核心设计原则

- **Graph as Capability**：每个业务能力是一个独立子图，通过路由图编排
- **State 驱动**：所有数据流经显式 State，可审计、可中断、可恢复
- **Tools 隔离**：工具层只做原子操作，业务逻辑在图节点中编排
- **存储无关**：工具层封装存储细节，将来从 JSON 换数据库只改工具层

## 零钱管理子图（当前实现）

### 流程图

```
START
  │
  ▼
[receive]  输入归一化
  │  短信原文 or 手动文本 → 统一格式
  │
  ▼
[parse]    LLM 提取账单
  │  文本 → {amount, merchant, category, note, time}
  │
  ▼
[record]   Tool: 写入 transactions.json
  │
  ▼
[check]    Tool: 计算余额 + 风险评级
  │  totalSpent / monthlyBudget
  │  输出 riskLevel: ok | warn | critical
  │
  ▼ (条件路由)
  ├─ ok (余额 > 50%)        → END（静默）
  ├─ warn (余额 20~50%)     → [advise] 温和提醒
  └─ critical (余额 < 20%)  → [advise] 严厉警告 + 退货建议
  │
  ▼
[advise]   LLM: 角色化生成建议
  │
  ▼
END
```

### State 定义

```javascript
const ExpenseState = Annotation.Root({
  // === 输入 ===
  rawInput: Annotation,          // 短信原文或用户手动文本
  source: Annotation,            // "sms" | "manual"

  // === 中间产物 ===
  transaction: Annotation({
    amount: Number,
    merchant: String,
    category: String,            // 餐饮 | 购物 | 交通 | 居住 | 娱乐 | 医疗 | 其他
    note: String,
    time: String,                // ISO 8601
  }),

  // === 统计结果 ===
  monthlyBudget: Annotation,     // 当月预算额度
  totalSpent: Annotation,        // 当月已消费
  remaining: Annotation,         // 剩余额度
  dailyBudget: Annotation,       // 剩余日均可用

  // === 风险 ===
  riskLevel: Annotation,         // "ok" | "warn" | "critical"

  // === 输出 ===
  advice: Annotation,            // AI 建议文本

  // === 异常 ===
  error: Annotation,
});
```

### 图组装

```javascript
import { StateGraph, END, START } from "@langchain/langgraph";
import { ExpenseState } from "./state.js";

const graph = new StateGraph(ExpenseState)
  .addNode("receive", receiveNode)
  .addNode("parse", parseNode)        // LLM
  .addNode("record", recordNode)      // Tool
  .addNode("check", checkNode)        // Tool
  .addNode("advise", adviseNode)      // LLM

  .addEdge(START, "receive")
  .addEdge("receive", "parse")
  .addEdge("parse", "record")
  .addEdge("record", "check")

  .addConditionalEdges("check", routeByRisk, {
    ok: END,
    warn: "advise",
    critical: "advise",
  })
  .addEdge("advise", END)

  .compile();
```

### 各节点职责

| 节点 | 类型 | 职责 | 关键逻辑 |
|---|---|---|---|
| receive | 纯函数 | 输入归一化 | 识别来源，去除短信服务商前缀 |
| parse | LLM | 文本转结构化 | few-shot prompt, 输出 JSON schema |
| record | Tool | 持久化 | 写入 transactions.json |
| check | Tool | 统计+评级 | 查询当月所有交易，算余额，分风险等级 |
| advise | LLM | 生成建议 | 根据 riskLevel 选择 prompt，注入角色人设 |

### 条件路由逻辑

```javascript
function routeByRisk(state) {
  const ratio = state.remaining / state.monthlyBudget;
  if (ratio > 0.5) return "ok";
  if (ratio >= 0.2) return "warn";
  return "critical";
}
```

graph TD
  A[START] --> B[collect: 收集用户风险偏好 + 资产现状]

## API 设计

### POST `/api/sms`

短信 webhook 入口。Android 短信转发 App 将短信 POST 到此。

```
Content-Type: application/json

{
  "sender": "95516",
  "body": "您尾号8888的借记卡于5月15日10:30消费38.00元，余额1,234.56元。【中国银联】",
  "receivedAt": "2026-05-15T10:31:00+08:00"
}

Response 200:
{
  "success": true,
  "transaction": { ... },
  "advice": "这笔咖啡记下了。这个月餐饮还剩 ¥420，节奏还行~"  // 仅在 warn/critical 时返回
}
```

### GET `/api/bills?year=2026&month=5`

查询某月账单。

```
Response 200:
{
  "year": 2026,
  "month": 5,
  "budget": 3000,
  "totalSpent": 1250,
  "remaining": 1750,
  "dailyBudget": 109.4,
  "transactions": [ ... ]
}
```

## 注意事项

- LangGraph JS 版 API 与 Python 版基本对齐，但文档和社区案例较少，开发时优先参考 [LangGraph JS API 文档](https://langchain-ai.github.io/langgraphjs/)
- 所有 LLM 调用集中在 `src/llm/client.js`，便于统一管理 retry、timeout、cost tracking
- JSON 文件存储虽然简单，但需要加简单的写锁防止并发写入脏数据
- checkout（中断+人工确认）能力是 LangGraph 的一大优势，大额消费场景将来可以加 `interrupt()` 断点
